Javascript & Ruby similarities and differences

March 2nd, 2015

Summary

Both Ruby and Javascript are object oriented programs or languages. This makes them very similar in terms of design, code, and logic. However, they are different. Javascript is used for Front-End Development, more specifically for the web browsers. It works with HTML, & CSS and gives websites motion and allows for interactivity. Things like drop down menus, sliders, an quizzes are all written in Javascript. While Ruby is a Back-End Development tool used for creating programs, the part of the code that you can’t see. Backend languages usually deal with servers and databases. So anytime you search for information or access your username and password for example, you are using a backend system. Let’s go over a few fundamental differences between Ruby and Javascript. Of course there are many more but this is a solid start. For simplicity, I will always put the Javascript code first and then the Ruby code in the examples below.

Ruby Hashes vs. JavaScript Objects

Creating a new object literal in Javascript is easy, and looks surprisingly similar to a hash literal in Ruby. You can use a JS object the same way you use a Ruby hash. If 'person' were a Ruby hash, we would say that it has the keys 'firstName', 'lastName', and 'age'. Since this is a JavaScript object, we refer to them as the properties of that object.

var person = { firstName: 'Frank', lastName: 'Hardy', age: 25 }; person = { "first_name" => 'Frank', "last_name" => 'Hardy', "age"=> 25}

In Ruby, the keys can be any data type: string, symbol, array, another hash, etc. This is not the case in JavaScript. The properties of an object are variables, and must conform to the same character requirements as other variables in JavaScript. Properties can still point to any data type value, including functions. Changing the property of an object is the same as changing the value of a variable, except we have to prefix the property with the object's name.

person.firstName = "Dora"; console.log('Full name: ' + person.firstName + ' ' + person.lastName);

You can use the same syntax to add properties that don't already exist, neat right?

var book = {}; book.title;
